in this set of documentation, we will learn how to setup galacticord, keep it up to date, and even run it as a backround service.

prerequisites

you'll want the following.
  • a VPS with at least 4gigs of ram.
  • your VPS mustrun Linux.
  • have git installed for cloning repositories. for exact instructions, check your Linux package managers documentation, but generally, most come with git already to go.
  • your discord bot token and bot owner IDs, check discord developer portalto get this information. note: your discord bot token will be under bot, and your client ID will be under oauth2.

      • optional, but recommended.

      • gemini API key, obtainable via .
        • a quick note, if you plan to actively use gemini for tts (gemini-2.5-flash-preview-tts) it is recommended you enable billing to get around the 10request per minute ratelimit. for exact pricing, check your teer status details.
      • youtube API key, obtainable via google cloud console, for searching youtube for the play command.
        • very important note: this is not for ytdlp in terms of searching. please use a proxy if you want to play videos using yt-dlp if you are on a VPS. if you're on a home computer, this is generally not required. note: you can try to use a cookies.txt file, but this is unlikely to work.
      • google cloud TTS API key, also obtainable through the google cloud console, however, requires billing to be enabled for your google cloud project.

        • easy setup instructions

        this will guide you through initial setup. 1. before you start, it is recommended that you make a separate folder for the bot, however, this is not recommended. in fact, git creates a folder for you, which means you don't necessarily have to do it yourself. nevertheless, if you still want to, run this command. mkdir galacticord
        1. clone the repository.
          • if you are in your main (~) (home) folder:
            • git clone https://github.com/g4p-studios/galacticord
          • if you are in a separate folder.
            • this usually only requires a dot at the end of your command. git clone https://github.com/g4p-studios/galacticord .
        2. run the setup script
          3. if git created a folder for you, cd into it now using cd foldername. E.G. cd galacticord, then run the following commands. chmod +x setup.sh && ./setup.sh
        3. once inside the setup script, click 1
          4. 1 means install. once you do that, give it about 15-30minutes to get everything installed. while this usually takes around 15-30 minutes, this highly depends on your internet connection and speed of your server or the computer you are using to run the bot. this will install everything required to run the piper voices, along with the Linux (RHVoice and Espeak) voices.
        4. after installation, it will ask you for your credentials. provide them when it asks
          5. note: if you do not have a certain credential, like an APi key, or you don't want to provide one, simply click enter. if you want to change this without running the setup script again, you can always do so in your .env file.
        when the setup script closes, it is likely you are ready to run the bot.

        how to run the bot for the first time

        when the bot is first setup, you have 2 options. running in the terminal while you're on the computer and in the backround.

        the terminal

        in the same foldr, type npm start. alternatively, you can also type ./setup.sh again, press enter, then press 2. this will allow you to see logs while also testing the bot.

        running in the backround.

        not only does the bot let you run it in the backround as a system service, it makes this quite easy. very important: before you do this, press control+c if you are running the bot in the terminal.
        1. type setup.sh then press enter.
        2. this time, press 3 and then press enter
        3. when it asks for a name, you can just press enter or type in a custom name for the process, E.G. gal
        assuming system is installed, the bot will attempt to make a system service. this will run the bot even on reboot. to start the service, type the following command. systemctl start servicename. if it's galacticord, than you would type systemctl start galacticord. your bot is now running in the backround.

        updating the bot

        often, updating the bot is as simple as running git pull . note: sometimes, you may need to run git stash, git pull, then git stash pop, depending on what gets updated, E.G. a script gets updated.

        very important

        depending on the feature and packages installed/updated, there may be times where you need to run git reset --hard. do not do this without running these commands.
        1. in the bots folder, run this command immediately!
          2. cp .env .env.bak this backs up your environment variables.
        2. in the same folder, cd over to the subfolder.
          3. cd data gcp tts_settings.json tts_settings.json.bak && cp server_config.json server_config.json.bak this backs up your server settings and any other server settings of servers the bot may be on.
        3. go back to the bots main folder and do systemctl stop servicename if you are running any system service with the bot.
        4. git reset --hard && git pull
        5. run the following commands to restore your settings.
          6. cp .env.bak .env &&cd data &&cp tts_settings.json.bak tts_settings.json && cp server_config.json.bak server_config.json
        you may now rerun the system service with systemctl start gal.
