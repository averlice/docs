Setting up Galacticord

In this set of documentation, we will learn how to setup Galacticord, keep it up to date, and even run it as a background service.

Prerequisites

You'll want the following:

  • A VPS with at least 4GB of RAM.
  • Your VPS must run Linux.
  • Git installed for cloning repositories. Check your Linux package manager's documentation (usually installed by default).
  • Node.js and npm, required to run the bot.
  • Your Discord bot token and Owner IDs from the Discord Developer Portal.
    Note: Your token is under "Bot", and Client ID is under "OAuth2".

Optional, but Recommended

  • Gemini API Key: Obtainable via Google AI Studio.
    If you plan to use Gemini for TTS (gemini-2.5-flash-preview-tts), enable billing to bypass the 10 requests per minute limit.
  • YouTube API Key: Obtainable via Google Cloud Console for the play command.
    This is not for yt-dlp searching. Use a proxy if running on a VPS. Home computers generally don't need this.
  • Google Cloud TTS API Key: Requires billing enabled on Google Cloud Console.

Easy Setup Instructions

  1. Create a folder (Optional): Git creates its own folder, but if you want to create one manually: 
    mkdir galacticord
  2. Clone the repository:

    If you are in your home folder (~):

    git clone https://github.com/g4p-studios/galacticord

    If you are in a specific folder (note the dot at the end):

    git clone https://github.com/g4p-studios/galacticord .
  3. Run the setup script:

    Enter the folder and run:

    cd galacticord && chmod +x setup.sh && ./setup.sh
  4. Select Installation:

    Inside the script, choose 1 (Install). This may take 15-30 minutes depending on your connection. This installs Piper, RHVoice, and Espeak voices.

  5. Provide Credentials:

    Follow the prompts for API keys. Press Enter to skip any you don't have. You can edit these later in the .env file.

How to Run the Bot

Option 1: In the Terminal

To see logs and test the bot, run:

npm start

Alternatively, run ./setup.sh and choose option 2.

Option 2: Running in the Background

Stop any running instance (Ctrl+C) first.

  1. Run ./setup.sh and choose option 3.
  2. Enter a name for the service (e.g., gal).
  3. Start the service: 
    sudo systemctl start galacticord

Updating the Bot

Usually, a simple git pull works. If you have local changes, use:

git stash && git pull && git stash pop

Important: Hard Reset Update

If a major update requires git reset --hard, follow these steps to prevent data loss:

  1. Backup your environment variables: 
    cp .env .env.bak
  2. Backup data settings: 
    cd data && cp tts_settings.json tts_settings.json.bak && cp server_config.json server_config.json.bak && cd ..
  3. Stop the service, reset, and pull: 
    sudo systemctl stop galacticord && git reset --hard && git pull
  4. Restore your settings: 
    cp .env.bak .env && cd data && cp tts_settings.json.bak tts_settings.json && cp server_config.json.bak server_config.json && cd ..

Rerun the service with sudo systemctl start gal.
